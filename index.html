Failure Modes & Fixes

Infinite loop
Supervisor keeps routing without progress.
Fix: max_iterations=6 guard in SupervisorAgent.
Agent crash
LLM call raises an exception mid-workflow.
Fix: try/except in each agent; error appended to state.errors.
No search key
Tavily API key missing or invalid.
Fix: SearchClient auto-falls back to mock documents.
Cascading errors
Multiple agent failures in a row.
Fix: Supervisor stops routing when len(errors) ≥ 3.

Final Answer (Writer Output)

Paper Title: AI Agents for Research Assistance: Capabilities, Evaluation, and Open Problems


Draft Abstract

This survey explores the growing role of AI agents in research assistance, emphasizing capabilities, evaluation methodologies, and existing challenges. AI agents enhance research workflows through automation of literature reviews, information synthesis, and analytical insights. However, quantifying their performance remains a significant challenge — many existing evaluations are neither comprehensive nor standardized. This paper reviews current advancements, highlights practical applications, discusses evaluation gaps, and proposes a framework for future benchmarking.


Section Outline

  1. Introduction — Define AI agents, trace their evolution in research contexts. Open problem: no standard classification of agent types for research tasks.
  2. Capabilities of AI Agents — Automation, synthesis, personalised assistance. Open problem: qualitative analysis remains underserved.
  3. State of the Art — Leading models, tools, algorithms; comparative performance. Open problem: no comprehensive cross-system benchmark exists.
  4. Practical Applications & Case Studies — Healthcare, social sciences, engineering. Pitfall: cherry-picking success stories over documented failures.
  5. Evaluation Methodologies — Quality metrics, precision/recall, user satisfaction. Open problem: standardised frameworks missing.
  6. Evaluation Gaps — Inconsistency, no longitudinal studies, neglect of user-centric metrics.
  7. Future Benchmarks — Comprehensive outcomes: effectiveness, efficiency, user satisfaction, scalability, cross-domain flexibility.

Exit Ticket

✓ Use Multi-Agent when…

  • Query requires distinct phases: gather → analyse → write
  • Traceability and auditability are required
  • Each phase needs a specialised system prompt
  • Latency and cost are not the primary constraints

✗ Avoid Multi-Agent when…

  • Query is simple or self-contained
  • Low latency is critical (chatbot, real-time API)
  • Cost per query must be minimised at scale
  • Coordination overhead exceeds the benefit
